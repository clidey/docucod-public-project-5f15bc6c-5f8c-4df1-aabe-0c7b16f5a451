# Installation Problems FAQ

This page addresses common challenges you may encounter while installing Task across different operating systems. Whether you're setting up your environment for the first time, facing compatibility issues, or struggling with permissions, this guide provides actionable steps and troubleshooting advice to help you get Task running smoothly.

---

## 1. General Installation Issues

### Why does Task fail to run after installation?
- Ensure that the Task binary is added to your system's `PATH` environment variable. Without this, your shell will not recognize the `task` command.
- On macOS and Windows, some default install locations like `~/.local/bin` or `~/bin` might not be included in `PATH`. Manually add them if using the install script.

### What platforms are supported by Task?
- Task supports Linux, macOS, and Windows on popular CPU architectures such as amd64 and arm64.
- Verify your platform against the [System Requirements](https://docs.taskfile.dev/getting-started/prerequisites-installation/system-requirements) to ensure compatibility.

### How do I verify that Task was installed correctly?
- Run `task --version` in your terminal or command prompt. Successful output confirms Task is accessible.
- If the command is not found, check your installation steps.

---

## 2. Permission and Access Errors

### Why do I get permission denied errors when installing Task?
- For system-level installs, you may need administrator or root privileges.
- Use `sudo` on Linux/macOS if installing into protected directories (e.g., `/usr/local/bin`).
- On Windows, run your terminal or PowerShell as Administrator for installs like Scoop or Chocolatey.

### How to fix permission issues with package managers?
- For Homebrew, make sure your user has write access to `/usr/local` or the appropriate Cellar directories.
- For Snap, the classic confinement must be enabled: `sudo snap install task --classic`.

## 3. Platform-Specific Installation Challenges

### Linux
- Ensure you have added the official Task repository correctly before installation using package managers like `dnf` or `apt`.
- If installing manually, verify binary compatibility with your distro and architecture.

### macOS
- When using Homebrew, choose the official tap to get the latest version and avoid outdated community forks.
- Add the binary directory to your `PATH` if installed in `~/.local/bin` or `~/bin`.

### Windows
- Use package managers like WinGet, Scoop, or Chocolatey for seamless installation.
- Confirm your shell environment recognizes installed bash-style commands if using npm or pip installations.
- Running PowerShell as Administrator often resolves permission or path recognition problems.

---

## 4. Installation Methods and Troubleshooting

### Using Official Package Managers
- Follow the setup instructions exactly for each package manager (dnf, apt, Homebrew, Snap, npm, WinGet).
- Repositories often require manual setup scripts before actual package installation.

### Binary Download
- Download the appropriate precompiled binary from [Task GitHub Releases](https://github.com/go-task/task/releases).
- Verify checksum with `task_checksums.txt` to avoid corrupted files.
- Place the binary in a directory included in your system's `PATH`.

### Install Script
- The script installs Task by downloading the latest binary automatically.
- You can specify installation directory and version tag.
- On macOS and Windows, remember to update your `PATH` if using default install locations like `~/.local/bin`.

### Building from Source
- Ensure you have a supported version of [Go](https://golang.org) installed.
- Use `go install github.com/go-task/task/v3/cmd/task@latest` to build and install.
- This method requires more setup but gives you complete control over the binary.
