## Reporting Bugs and Feature Requests

Your feedback helps improve Task for everyone. Please follow these guidelines to streamline support and resolution.

### How to Report Bugs

1. **Verify the Issue:** Check if the problem is documented in the FAQ or existing GitHub issues.
2. **Gather Details:** Include Task version (`task --version`), operating system, your Taskfile snippet, and exact commands run.
3. **Create a Minimal Reproducible Example:** Simplify your Taskfile to isolate the issue.
4. **Open an Issue:** Submit your bug report at [Task GitHub Issues](https://github.com/go-task/task/issues) with all collected information.

### Requesting Features

- Use GitHub Discussions or Issues to propose new features or enhancements.
- Provide clear use cases and expected behavior to help maintainers understand your needs.

<Tip>
Be respectful and clear when communicating issues or feature requests. The maintainers and community volunteers dedicate their time to improve Task.
</Tip>

## Best Practices for Seeking Help

- **Search Existing Resources:** Before asking, search existing documentation, FAQs, and issues for solutions.
- **Use Verbose Mode:** Add `-v` or `--verbose` to your Task commands to get detailed diagnostic outputs when asking for help.
- **Share Logs and Error Messages:** Include full error outputs in your questions or bug reports.
- **Specify Context:** Mention your platform, Task version, and any relevant environment settings.
